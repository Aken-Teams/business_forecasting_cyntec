## 系統概述

### 系統名稱
**FORECAST 數據處理系統** - 智能預測數據整合與分析平台

### 系統定位
企業級業務預測數據處理系統，專為處理 ERP 淨需求數據、Forecast 預測數據和在途運輸數據而設計，提供智能化的數據整合、清理和預測填寫功能。

### 技術棧
- **後端框架**: Flask 3.0.3 (Python Web Framework)
- **數據處理**: pandas 2.2.2, numpy 1.26.4
- **Excel操作**: openpyxl 3.1.5, xlrd 2.0.1
- **前端技術**: HTML5, CSS3, JavaScript (ES6+)
- **UI庫**: Font Awesome 6.0.0
- **Web伺服器**: Nginx (生產環境)

### 核心特性
- ✅ 超高速批量處理 (性能提升20倍)
- ✅ 智能數據累加邏輯
- ✅ 可視化映射配置介面
- ✅ 多數據源整合 (ERP + Forecast + Transit)
- ✅ 自動日期計算與匹配
- ✅ 瀏覽器相容性檢測

## 核心功能

### 1. 文件上傳管理
**功能描述**: 支援三種類型 Excel 文件的上傳與驗證

#### 支援的文件類型
1. **ERP淨需求文件** (`erp_data.xlsx`)
   - 包含客戶需求、料號、排程出貨日期等信息
   - 必須包含「客戶簡稱」欄位

2. **Forecast預測文件** (`forecast_data.xlsx`)
   - 包含週別預測數據
   - 包含日期範圍和供應數量信息

3. **在途文件** (`transit_data.xlsx`)
   - 12個標準欄位結構
   - 包含船期、ETA、客戶信息等

#### 文件驗證
- 格式驗證 (僅支援 .xlsx/.xls)
- 欄位結構驗證
- 數據完整性檢查
- 即時上傳狀態回饋

### 2. 數據清理 (Data Cleanup)
**功能描述**: 自動清理 Forecast 文件中的特定數據，避免歷史數據干擾

#### 清理規則
1. **供應數量清理**
   - 條件: K欄位 = "供應數量"
   - 操作: 清空 L~AW 欄位 (第12-49列)

2. **庫存數量清理**
   - 條件: I欄位包含 "庫存數量"
   - 操作: 清空下一列的 I 欄位

#### 技術特點
- 使用 openpyxl 保持原始格式
- 批量處理提升效率
- 保留 Excel 文件結構和樣式

### 3. 映射表配置 (Mapping Configuration)
**功能描述**: 可視化配置客戶簡稱與業務參數的映射關係

#### 配置項目
1. **客戶需求地區**: 地區代碼匹配
2. **排程出貨日期斷點**: 週別計算基準 (例: 禮拜四)
3. **ETD** (Estimated Time of Departure): 預估離港時間
4. **ETA** (Estimated Time of Arrival): 預估到貨時間

#### 數據同步
- JSON 格式存儲 (`mapping_data.json`)
- Excel 格式同步 (`mapping表.xlsx`)
- 自動讀取和更新機制

#### 介面特性
- 表格式編輯介面
- 即時保存與驗證
- 批量配置支援

### 4. ERP 與在途數據整合
**功能描述**: 根據映射表整合 ERP 和在途數據

#### ERP 整合邏輯
```
輸入: ERP文件 + 映射表
處理:
  1. 讀取客戶簡稱
  2. 匹配映射表數據
  3. 添加新欄位:
     - 客戶需求地區
     - 排程出貨日期斷點
     - ETD
     - ETA
  4. 按排程出貨日期排序
輸出: integrated_erp.xlsx
```

#### 在途數據整合邏輯
```
輸入: 在途文件 (12欄) + 映射表
匹配鍵: E欄位 (客戶簡稱) <-> 映射表 A欄位
處理:
  1. 讀取 E 欄位客戶簡稱
  2. 查找映射表匹配項
  3. 添加4個新欄位 (索引12-15):
     - 客戶需求地區
     - 排程出貨日期斷點
     - ETD
     - ETA_mapping
輸出: integrated_transit.xlsx (16欄)
```

#### 日期標準化
- 統一處理多種日期格式
  - `2025/10/01` (斜線格式)
  - `2025-10-01` (連字符格式)
  - `20251001` (純數字格式)
  - pandas Timestamp 對象
  - datetime 對象
- 自動解析和轉換
- 錯誤處理和日誌記錄

### 5. FORECAST 超高速處理
**功能描述**: 批量處理預測數據填寫，性能提升20倍

#### 核心處理流程
```
1. 數據載入階段
   - 載入 cleaned_forecast.xlsx
   - 載入 integrated_erp.xlsx
   - 載入 integrated_transit.xlsx (可選)
   - 建立索引加速查找

2. 數據塊識別
   - 掃描 A 欄位 (客戶料號)
   - 掃描 D 欄位 (客戶需求地區)
   - 識別連續數據塊
   - 記錄起始和結束行

3. ERP 數據處理
   對每個數據塊:
   ├─ 匹配鍵: 客戶料號 + 客戶需求地區
   ├─ 查找 ERP 記錄
   ├─ 計算目標日期
   │  ├─ 解析排程出貨日期
   │  ├─ 計算週別範圍 (根據斷點)
   │  └─ 根據 ETA 計算目標日期
   ├─ 轉換數值 (淨需求 * 1000)
   ├─ 查找填寫位置
   └─ 添加到批量修改列表

4. Transit 數據處理
   對每個數據塊:
   ├─ 匹配鍵: 客戶需求地區 + 客戶料號
   │  (Transit M欄 + F欄)
   ├─ 查找 Transit 記錄
   ├─ 讀取 H 欄位數值
   ├─ 讀取 I 欄位 ETA
   ├─ 轉換數值 (H值 * 1000)
   ├─ 解析 ETA 日期
   ├─ 查找填寫位置
   └─ 添加到批量修改列表 (累加)

5. 批量寫入階段
   - 一次性應用所有修改
   - 支援累加邏輯
   - 統計累加情況
   - 填充 D 欄位空白單元格 (保持 Excel 篩選功能)

6. 結果保存
   - 保存為 forecast_result.xlsx
   - 保留原始格式和樣式
```

#### 日期計算邏輯

**週別計算**:
```python
# 根據排程出貨日期斷點計算週別範圍
斷點 = "禮拜四"
排程日期 = "2025/10/15" (假設是週三)

計算過程:
1. 找到這週的週四: 2025/10/16
2. 週別範圍: 
   - 週開始 (週五): 2025/10/10
   - 週結束 (週四): 2025/10/16
```

**ETA 日期計算**:
```python
基準週別: 2025/10/10 ~ 2025/10/16
標準週: 禮拜天 ~ 禮拜六

ETA = "下下週二"
計算過程:
1. 找到基準週的禮拜六: 2025/10/11 (週六)
2. 找到對應的禮拜天: 2025/10/06 (週日)
3. 加14天 (下下週): 2025/10/20 (週日)
4. 找到週二: 2025/10/22

支援格式:
- "本週X": 當前標準週的星期X
- "下週X": 下一標準週的星期X  
- "下下週X": 下下標準週的星期X
```

#### 性能優化技術
1. **索引優化**
   - pandas DataFrame 索引
   - 字典鍵匹配 (`客戶料號_客戶需求地區`)
   - O(1) 查找複雜度

2. **批量操作**
   - 預先計算所有修改
   - 一次性寫入 Excel
   - 減少 I/O 操作

3. **內存管理**
   - 工作簿保持在內存
   - 避免重複讀寫
   - 延遲保存策略

4. **累加邏輯**
   - 檢測同位置多次寫入
   - 自動累加數值
   - 避免數據覆蓋

### 6. 結果下載
**功能描述**: 提供處理結果文件的下載功能

#### 可下載文件
- `forecast_result.xlsx`: FORECAST 最終處理結果

#### 文件特性
- 保留原始 Excel 格式
- 支援 Excel 篩選功能
- 完整數據填寫

### 各階段詳細說明

#### 階段1: 文件上傳
**目的**: 收集處理所需的原始數據

**輸入**:
- ERP淨需求文件 (Excel)
- Forecast預測文件 (Excel)
- 在途文件 (Excel)

**處理**:
1. 前端驗證文件格式
2. 上傳到服務器 `uploads/` 目錄
3. 後端驗證文件結構
4. 讀取並顯示基本信息

**輸出**:
- `uploads/erp_data.xlsx`
- `uploads/forecast_data.xlsx`

### 核心算法詳解

#### 1. 日期標準化算法
```python
def normalize_date(date_value):
    """
    統一處理各種日期格式
    支援:
    - datetime/pd.Timestamp 對象
    - 字串格式 (多種格式)
    - 數字格式
    """
    # 空值處理
    if pd.isna(date_value) or date_value is None:
        return None
    
    # 已是 datetime 對象
    if isinstance(date_value, (datetime, pd.Timestamp)):
        return date_value
    
    # 字串解析
    if isinstance(date_value, str):
        formats = [
            "%Y/%m/%d",   # 2025/10/01
            "%Y-%m-%d",   # 2025-10-01
            "%m/%d/%Y",   # 10/01/2025
            "%d/%m/%Y",   # 01/10/2025
            "%Y%m%d",     # 20251001
        ]
        for fmt in formats:
            try:
                return datetime.strptime(date_str, fmt)
            except:
                continue
        
        # 使用 pandas 自動解析
        try:
            return pd.to_datetime(date_str)
        except:
            return None
    
    # 其他類型使用 pandas 轉換
    try:
        return pd.to_datetime(date_value)
    except:
        return None
```

#### 2. 週別計算算法
```python
def calculate_week_range(schedule_date, breakpoint):
    """
    根據排程出貨日期和斷點計算週別範圍
    
    例子:
    schedule_date = 2025/10/15 (週三)
    breakpoint = "禮拜四"
    
    計算過程:
    1. 禮拜四 = weekday 3
    2. 週三 = weekday 2
    3. 距離週四 = (3 - 2) % 7 = 1 天
    4. week_end = 2025/10/16 (週四)
    5. week_start = 2025/10/10 (週五，前6天)
    
    返回: (2025/10/10, 2025/10/16)
    """
    weekday_map = {
        '禮拜一': 0, '禮拜二': 1, '禮拜三': 2, '禮拜四': 3,
        '禮拜五': 4, '禮拜六': 5, '禮拜日': 6
    }
    
    week_end_day = weekday_map.get(breakpoint, 3)
    days_to_end = (week_end_day - schedule_date.weekday()) % 7
    week_end = schedule_date + timedelta(days=days_to_end)
    week_start = week_end - timedelta(days=6)
    
    return week_start, week_end
```

#### 3. ETA 日期計算算法
```python
def calculate_eta_date(eta, week_start, week_end):
    """
    根據 ETA 文字計算目標日期
    
    流程:
    1. 找到週別對應的標準週 (禮拜天~禮拜六)
    2. 根據 ETA 文字偏移週數
    3. 找到目標星期幾
    
    例子:
    week_start = 2025/10/10 (週五)
    week_end = 2025/10/16 (週四)
    eta = "下下週二"
    
    計算:
    1. 找到 week_end 對應的禮拜六:
       days_to_saturday = (5 - 3) % 7 = 2
       saturday = 2025/10/18
    2. 找到禮拜天:
       sunday = 2025/10/12
    3. 下下週禮拜天:
       next_next_sunday = 2025/10/12 + 14 = 2025/10/26
    4. 找到禮拜二:
       tuesday = 2025/10/26 + 2 = 2025/10/28
    
    返回: 2025/10/28
    """
    # 找到標準週的禮拜六
    days_to_saturday = (5 - week_end.weekday()) % 7
    saturday = week_end + timedelta(days=days_to_saturday)
    sunday = saturday - timedelta(days=6)
    
    # 解析 ETA 文字
    if '下下週' in eta:
        target_sunday = sunday + timedelta(days=14)
    elif '下週' in eta:
        target_sunday = sunday + timedelta(days=7)
    else:  # 本週
        target_sunday = sunday
    
    # 提取星期幾
    weekday_map = {'一': 0, '二': 1, '三': 2, '四': 3, 
                   '五': 4, '六': 5, '日': 6}
    for char, day in weekday_map.items():
        if char in eta:
            return target_sunday + timedelta(days=day)
    
    return None
```

#### 4. 填寫位置查找算法
```python
def find_target_position(target_date_str, start_row, end_row):
    """
    在 Forecast 文件中查找目標日期的填寫位置
    
    邏輯:
    1. 掃描 K~AW 欄位 (第11-49列)
    2. 檢查每個欄位的日期範圍
       - start_date (start_row + 1)
       - end_date (start_row + 2)
    3. 判斷 target_date 是否在範圍內
    4. 找到對應的 "供應數量" 行
    
    例子:
    target_date_str = "20251028"
    K欄位範圍: 20251021 - 20251027 (不匹配)
    L欄位範圍: 20251028 - 20251103 (匹配!)
    
    查找供應數量行:
    掃描 start_row 到 start_row+18
    找到 K欄位 = "供應數量" 的行
    
    返回: (col_idx=12, row_idx=對應行號)
    """
    for col_idx in range(10, min(49, len(forecast_df.columns))):
        # 讀取日期範圍
        start_date_row = start_row + 1
        end_date_row = start_row + 2
        
        start_date = forecast_df.iloc[start_date_row, col_idx]
        end_date = forecast_df.iloc[end_date_row, col_idx]
        
        # 檢查格式和範圍
        if pd.notna(start_date) and pd.notna(end_date):
            start_str = str(int(float(start_date)))
            end_str = str(int(float(end_date)))
            
            if len(start_str) == 8 and len(end_str) == 8:
                if start_str <= target_date_str <= end_str:
                    # 找到目標欄位，查找供應數量行
                    for row_idx in range(start_row, start_row + 18):
                        k_value = forecast_df.iloc[row_idx, 10]
                        if pd.notna(k_value) and str(k_value) == "供應數量":
                            # 轉換為 openpyxl 索引
                            openpyxl_row = row_idx + 2
                            return col_idx, openpyxl_row
    
    return None, None
```

#### 5. 批量累加算法
```python
def add_change_to_batch(col_idx, row_idx, value):
    """
    添加修改到批量列表，支援累加
    
    邏輯:
    1. 檢查是否已有相同位置的修改
    2. 如果有，累加值
    3. 如果沒有，添加新修改
    
    這樣可以處理多筆 ERP/Transit 數據填入同一個位置的情況
    """
    # 查找現有修改
    for change in pending_changes:
        if change['col_idx'] == col_idx and change['row_idx'] == row_idx:
            # 找到相同位置，累加值
            change['value'] += value
            return True
    
    # 沒有找到，添加新修改
    pending_changes.append({
        'col_idx': col_idx,
        'row_idx': row_idx,
        'value': value
    })
    return True

def apply_all_changes():
    """
    批量應用所有修改到 Excel 工作簿
    """
    for change in pending_changes:
        col_idx = change['col_idx']
        row_idx = change['row_idx']
        value = change['value']
        
        # 寫入 Excel (openpyxl)
        cell = worksheet.cell(row=row_idx, column=col_idx + 1)
        cell.value = value
```

### Nginx 配置要點
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 根目錄
    root /path/to/business_forecasting;
    
    # 靜態資源
    location /static/ {
        alias /path/to/business_forecasting/static/;
        expires 1h;  # CSS/JS 緩存1小時
    }
    
    # 上傳文件
    location /uploads/ {
        alias /path/to/business_forecasting/uploads/;
        internal;  # 內部訪問
    }
    
    # 處理結果
    location /processed/ {
        alias /path/to/business_forecasting/processed/;
        internal;
    }
    
    # 應用代理
    location / {
        proxy_pass http://127.0.0.1:12026;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 不緩存動態內容
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        add_header Pragma "no-cache";
        add_header Expires "0";
    }
    
    # 最大上傳文件大小
    client_max_body_size 100M;
}
```
