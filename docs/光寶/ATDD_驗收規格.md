## Story 1: 完整 Forecast 處理流程

**作為** 光寶的 Forecast 管理人員
**我想要** 上傳 Forecast + ERP 檔案後一鍵處理
**以便** 取得填入 ERP 數量的 Forecast 結果檔

### 驗收條件

#### AC-1.1: 端到端流程正確完成

**前置條件:**
- 已配置 21 筆客戶映射（涵蓋 Type 11 與 Type 32）
- 上傳 1 個 Forecast 檔案（Plant=2680）
- 上傳 1 個 ERP 檔案（含 2680 的排程資料）

**執行步驟:**
1. 步驟一：清理 Forecast（Commit row 清零）
2. 步驟二：ERP 映射（填入地區、斷點、ETD/ETA）
3. 步驟三：執行 Forecast 處理

**驗收標準:**
| 檢查項目 | 預期結果 |
|---------|---------|
| 產出檔案 | `forecast_2680_A33.xlsx` |
| Commit row 數值 | 非零值 = 淨需求 × 1000 的正確累加 |
| 日期欄位對應 | 每個填入值都在正確的 Daily/Weekly/Monthly 欄位 |
| 已分配追蹤 | ERP 中被填入的行標記為 "✓" |
| 統計數據 | 顯示正確的「填入 N 筆，跳過 M 筆」 |

#### AC-1.2: 處理失敗時的回報

**前置條件:**
- Forecast 檔案格式錯誤（無 Daily+Weekly+Monthly sheet）

**驗收標準:**
- 系統回報明確錯誤訊息
- 不產出損壞的結果檔
- 其他正常檔案不受影響

## Story 5: 多 Plant 合併處理

**作為** 光寶的 Forecast 管理人員
**我想要** 將多個 Plant 合併成一個檔案處理
**以便** 簡化作業流程

### 驗收條件

#### AC-5.1: 合併檔案結構正確

| 檢查項目 | 預期結果 |
|---------|---------|
| Row 1 | 欄位 headers（含 Plant, Buyer Code, 原始 header）|
| Column A | Plant code（每行一個值）|
| Column B | Buyer code（每行一個值）|
| 原始欄位 | 全部右移 2 位 |
| 日期對齊 | 不同 Plant 相同日期 → 同一欄 |

#### AC-5.2: 合併模式的 Plant Daily 限制

**場景:** 合併後 Daily 欄位包含 3/7~4/8（聯集），但 429E 原始只到 4/6

| 目標日期 | Plant | 合併模式結果 | 原因 |
|---------|-------|------------|------|
| 4/7 | 429E | 不匹配 Daily → GAP/Weekly fallback | plant_daily_limit=4/6 |
| 4/7 | 2680 | 不匹配 Daily（2680 日期欄位中無 4/7） | 正常查找 |
| 4/8 | 2680 | 匹配 Daily | plant_daily_limit=4/8 |

## Story 6: Transit 在途資料處理

**作為** 光寶的 Forecast 管理人員
**我想要** Transit 在途數量也填入 Forecast
**以便** Commit row 反映完整的供應狀況

### 驗收條件

#### AC-6.1: Transit 端到端處理

**前置條件:**
- Transit 檔案包含 ETA 日期和 Qty
- 客戶映射已完成 Transit 的地區映射

**驗收標準:**
| 檢查項目 | 預期結果 |
|---------|---------|
| 數量轉換 | Qty × 1000 = 填入值 |
| 日期對應 | ETA 日期直接查找欄位（Daily→Weekly→Monthly）|
| 處理順序 | Transit 在 ERP 之前填入 |
| 累加行為 | Transit + ERP 同 cell → 值累加 |
| 已分配追蹤 | Transit 填入後標記 "✓" |

#### AC-6.2: 需在途開關

| 映射配置 | Transit 行為 |
|---------|------------|
| requires_transit=True | 正常處理 Transit |
| requires_transit=False | 跳過該 Plant 的 Transit |

## 已知限制與例外

| 項目 | 說明 |
|-----|------|
| 光寶 user_id | 硬編碼為 6，未來如需多用戶需重構 |
| 月份範圍 | Monthly 最多到 BQ (col 69)，超出無法填入 |
| 公式保留 | 合併模式會嘗試轉換公式，但複雜公式可能失敗 |
| 斷點文字 | 僅支援「禮拜X」「週X」「星期X」三種格式 |
| 日期文字 | 僅支援「X週Y」「X禮拜Y」格式，不支援「X星期Y」 |
