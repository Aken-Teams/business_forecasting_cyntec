## 一、會議目的

針對廣達 (Quanta) 與和碩 (Pegatron) 兩大客戶之 FORECAST 數據處理需求進行彙整與確認，確定系統功能範圍與處理規則。

---

## 二、需求討論內容

### 2.1 PJOMR006 淨需求處理邏輯

需求方提出 PJOMR006 淨需求檔案之欄位驗證與 KEY 值比對規則：

| 項目 | 廣達 | 和碩 |
|------|------|------|
| 必要欄位 | 客戶簡稱、客戶 PO、客料、淨需求、送貨地點、排程出貨日、倉庫 | 同廣達 |
| KEY 值 | 客戶簡稱 + 客料 | 客戶簡稱 + Line 客戶採購單號 + 客料 |

**決議**: 系統需依客戶別使用不同 KEY 值進行數據比對。

### 2.2 客戶 FCST 格式與處理

需求方說明 FCST 報表格式需符合各客戶模板，並確認以下規則：

- **和碩 FCST KEY 值**: Plant + MRP ID + PN Model（如：3A32 + A00Y + 0703-00AG000）
- **FCST 結構**: 每個產品佔 8 列（FORECAST、OTW QTY、ETA DATE、ETA QTY、Balance1、Balance2、WOS 等）
- **和碩特殊需求**: 需支援多廠區 (Plant) 上傳，並可依廠區合併/拆分下載
- **FCST 回覆方式**: 以排程出貨日為 KEY，依週別 Mapping 後回填 ETA QTY

**決議**: 系統需支援和碩多廠區處理邏輯，FCST 回填以 ETA QTY 列為目標。

### 2.3 ETA Mapping 規則

需求方提供和碩各送貨地點之 ETA Mapping 對應表：

| 送貨地點 | 廠區 (Plant) | 排程出貨日斷點 | ETD | ETA |
|----------|-------------|-------------|-----|-----|
| MAINTEK-新寧 | 3A32 / 3A33 / 3A35 / 3A36 / 3A3B | 禮拜三 | 下週二 | 下下週二 |
| 旭碩-飛力達 | 6071 / 6072 / 6075 | 禮拜三 | 下週一 | 下下週一 |
| 和碩-印尼 | 5491 | 禮拜三 | 下週二 | 下下週二 |
| 和碩-越南 | 5622 / 5691 / 5695 / 5696 / 5699 | 禮拜三 | 下週二 | 下下週二 |
| 和碩-新寧 | 3A39 | 禮拜三 | 下週二 | 下下週二 |
| 和碩-龜山 | 1104 / 1105 | 禮拜一 | 禮拜五 | 下週一 |
| 和碩墨西哥 | 58A1 | 禮拜三 | 禮拜二 | 下週二 |

- **ETA Mapping KEY 值**: 客戶簡稱 + Plant（客戶廠區）
- Mapping 表需支援動態更新 ETA，並提供 CRUD 維護功能

**決議**: 系統需建立 Mapping 管理介面，支援依客戶/送貨地點/廠區配置排程斷點、ETD 及 ETA。

### 2.4 在途清單處理

需求方說明在途清單之處理規則：

- **和碩在途 KEY 值**: Line 客戶採購單號 + Ordered Item
- 在途清單需與 FCST 進行比對，將在途數量回填至對應週別
- 資料來源包含 TW（台灣）及 WX（無錫）兩地出貨

**決議**: 系統需支援在途數據整合，依 KEY 值比對後回填至 FCST。

### 2.5 系統整合功能

需求方對系統操作流程提出以下要求：

| 功能需求 | 說明 |
|----------|------|
| 客戶下拉選單 | 系統提供客戶選擇，依客戶切換對應處理邏輯 |
| 格式驗證 | PJOMR006 及 FCST 依客戶別執行格式驗證 |
| 流程步驟控制 | 各處理階段需依序執行，上一步完成才可進入下一步 |
| 和碩廠區下載 | 和碩需支援依廠區分別下載處理結果 |

**決議**: 系統採五階段步驟式流程設計（上傳 → 清理 → Mapping 整合 → Forecast 處理 → 下載）。
